Project 02 — Digital Input Reliability

Debounced Button LED Control

This project improves a basic button-controlled LED circuit by adding software debouncing. The aim is to make sure one physical button press produces one clean action, even when the mechanical switch creates noisy transitions.

Components

  • Raspberry Pi Pico or compatible microcontroller
  • Pushbutton connected to GP15
  • Onboard LED or external LED
  • 10k ohm pull-down resistor
  • CircuitPython firmware

How It Works

Button bounce happens when the internal metal contacts of a switch vibrate briefly during a press or release. Without filtering, the microcontroller may read several HIGH and LOW changes from one press. The debounce algorithm waits until the input has remained stable for a short time before toggling the LED state.

Core Logic

import board import digitalio import time button = digitalio.DigitalInOut(board.GP15) button.direction = digitalio.Direction.INPUT led = digitalio.DigitalInOut(board.LED) led.direction = digitalio.Direction.OUTPUT last_reading = False stable_state = False last_button_state = False led_state = False last_change_time = time.monotonic() debounce_delay = 0.05 while True: reading = button.value if reading != last_reading: last_change_time = time.monotonic() if time.monotonic() - last_change_time > debounce_delay: if reading != stable_state: stable_state = reading if stable_state and not last_button_state: led_state = not led_state led.value = led_state last_button_state = stable_state last_reading = reading time.sleep(0.01)

What I Learned

The project showed that stable embedded systems are not only about wiring the circuit correctly. Software must also handle real hardware behavior, especially when mechanical components produce noisy signals.
