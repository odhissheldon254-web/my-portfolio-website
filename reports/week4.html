Week 04 — Asynchronous Logic

Interrupts and Timers

In Week 4, we moved beyond basic sequential execution by diving into hardware interrupts and timers. Relying on empty delay() functions wastes CPU cycles; interrupts allow an embedded system to react instantly to external events while performing other critical background work.

Learning Objectives

  • Understanding Interrupt Service Routines (ISRs).
  • Configuring Hardware Timers (Timer0, Timer1 on AVR).
  • Debouncing switches using timer-based logic instead of delays.
  • Best practices for volatile variables and ISR speed optimization.

Key Insights

One major takeaway was learning that an ISR must execute as quickly as possible. Performing heavy math or operations like serial printing inside an ISR can cause the microcontroller to crash or miss other critical timing events. We explored how to use flags to signal the main loop to perform the heavy lifting.

// Example: Hardware Interrupt with Flag Logic const int interruptPin = 2; volatile bool eventTriggered = false; void setup() { Serial.begin(9600); pinMode(interruptPin, INPUT_PULLUP); attachInterrupt(digitalPinToInterrupt(interruptPin), handleEvent, FALLING); } void loop() { if (eventTriggered) { Serial.println("Interrupt detected! Processing..."); // Reset flag eventTriggered = false; } } // This ISR runs incredibly fast void handleEvent() { eventTriggered = true; }
