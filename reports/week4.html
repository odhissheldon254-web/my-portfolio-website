Week 04 — Pico & Sensors

Raspberry Pi Pico Programming and Sensor Integration

Week 4 focused on moving from simple simulations into Raspberry Pi Pico programming and sensor-based embedded systems. The work combined C, CircuitPython, hardware input/output, sensor interfacing, interrupts, and the communication methods used to connect microcontrollers to external devices.

Learning Objectives

  • Program the Raspberry Pi Pico using both C and CircuitPython.
  • Connect external sensors to a microcontroller and read their signals.
  • Use interrupts to respond to hardware events without constantly polling.
  • Understand how UART, SPI, and I2C support sensor and peripheral communication.
  • Build confidence moving from simulated circuits to physical embedded hardware.

Sensor Integration

The Pico acts as the controller that reads sensor signals, processes them in software, and then produces a useful output. Sensor integration requires correct wiring, correct voltage levels, and code that converts raw readings into meaningful information. This week also introduced the idea that a sensor project is both a hardware problem and a software problem.

Interrupts

Interrupts allow a microcontroller to respond immediately when something important happens, such as a button press or a sensor trigger. Instead of wasting CPU time checking an input forever, the main program can continue running and an interrupt service routine can set a flag when attention is needed. A key lesson was that interrupt routines should stay short and avoid heavy work such as long calculations or serial printing.

// Interrupt pattern: keep the ISR short and process later const int sensorPin = 2; volatile bool eventTriggered = false; void setup() { Serial.begin(9600); pinMode(sensorPin, INPUT_PULLUP); attachInterrupt(digitalPinToInterrupt(sensorPin), handleEvent, FALLING); } void loop() { if (eventTriggered) { Serial.println("Interrupt detected! Processing..."); // Reset flag eventTriggered = false; } } // This ISR runs incredibly fast void handleEvent() { eventTriggered = true; }

Communication Notes

UART is useful for simple serial communication and debugging. I2C allows several addressed devices to share two lines, SDA and SCL, which is useful for many sensors and displays. SPI uses separate data and clock lines plus chip select, making it faster and suitable for peripherals such as displays, SD card modules, and radio modules.

Reflection

This week made the embedded systems workflow clearer: select a sensor, wire it correctly, choose the right communication method, then write code that reacts reliably. Interrupts are especially important because they make a system more responsive without forcing every task to run in a blocking loop.
