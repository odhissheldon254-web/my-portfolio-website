Week 06 — Communication & ESP32

Communication Protocols and Introduction to ESP32

This week focused on communication protocols used in embedded systems and the transition toward ESP32 development. The work covered UART, SPI, I2C, Wi-Fi, ESP-IDF setup, and a practical debounce assignment connected to reliable digital input.

Learning Objectives

  • Understand why mechanical pushbuttons can produce bouncing signals.
  • Compare synchronous and asynchronous serial communication.
  • Identify when UART, SPI, I2C, and Wi-Fi are useful in embedded systems.
  • Prepare the ESP32 development environment by installing ESP-IDF and the Arduino IDE board package.
  • Understand why mechanical pushbuttons can produce bouncing signals.
  • Implement software debounce so one press creates one clear action.

Assignment: Debounced LED Toggle

The hardware setup uses a pushbutton connected to GP15 with a 10k ohm pull-down resistor, and the onboard LED as the output. A raw button read can toggle the LED many times from a single press because the metal contacts inside the switch vibrate briefly. The debounce algorithm waits until the reading has stayed stable for a short interval before accepting the press.

import board import digitalio import time button = digitalio.DigitalInOut(board.GP15) button.direction = digitalio.Direction.INPUT led = digitalio.DigitalInOut(board.LED) led.direction = digitalio.Direction.OUTPUT last_reading = False stable_state = False last_button_state = False led_state = False last_change_time = time.monotonic() debounce_delay = 0.05 while True: reading = button.value if reading != last_reading: last_change_time = time.monotonic() if time.monotonic() - last_change_time > debounce_delay: if reading != stable_state: stable_state = reading if stable_state and not last_button_state: led_state = not led_state led.value = led_state last_button_state = stable_state last_reading = reading time.sleep(0.01)

Communication Protocols

UART is asynchronous, so it does not use a shared clock. It sends data with start and stop bits and is useful for simple serial links. SPI is synchronous and fast, using MOSI, MISO, SCLK, and a chip-select line to communicate with devices such as SD card modules and displays. I2C is also synchronous, but it uses only SDA and SCL lines plus device addresses, making it practical when many low-speed sensors share one bus.

Wi-Fi extends embedded communication beyond wires. A board can connect as a client to an existing network or act as a small server for web-based control and monitoring.

ESP32 Setup

ESP32 introduces built-in Wi-Fi and Bluetooth support, making it useful for connected embedded systems. The setup task was to install ESP-IDF, review the ESP32 development workflow, and add the Espressif board manager package in Arduino IDE before the next class.

Reflection

The main lesson was that embedded systems need reliable input handling and the correct communication method for each job. Pull-down resistors and debounce logic keep button input stable, while UART, SPI, I2C, and Wi-Fi each solve different data-transfer problems.
