Week 02 — Foundations

Hardware & Software Integration: C Programming

The second week of the program was dedicated to mastering C programming, the "mother language" of embedded systems. While Python is great for prototyping, C provides the low-level control and efficiency required for real-time instrument development.

Learning Objectives

  • Fundamentals of C Syntax and Logic.
  • Advanced Data Structures: Structs and Unions.
  • Dynamic Memory Management using malloc and free.
  • Understanding Memory Layout (Stack vs Heap).

Deep Dive: Memory Management

Unlike high-level languages with garbage collection, C requires the developer to manually manage memory. We explored how malloc allocates chunks of memory on the heap and why it is critical to use free to prevent memory leaks—a common cause of failure in long-running embedded systems.

We also studied the difference between Structs (where members have their own memory) and Unions (where members share the same memory), which is essential for memory-constrained microcontrollers.

Assignment: Student Management System

This project demonstrates the use of dynamic memory allocation and complex data structures. It allows for flexible data storage for students with different academic profiles.

// Student Management System in C #include <stdio.h> #include <stdlib.h> #include <string.h> typedef enum { UNDERGRAD, POSTGRAD } Level; typedef union { float cgpa; char thesis[100]; } AcademicData; typedef struct { char name[50]; int age; Level level; AcademicData data; } Student; int main() { int n; printf("Enter number of students: "); scanf("%d", &n); Student *list = (Student *)malloc(n * sizeof(Student)); if (!list) return 1; // Data Entry Logic... for(int i=0; i<n; i++) { printf("\nStudent %d Name: ", i+1); scanf("%s", list[i].name); // ... entries for age and level ... } // Displaying results... printf("\n--- Registry Data ---\n"); // ... print logic ... free(list); return 0; }
