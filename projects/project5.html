Capstone Project

AIROS: Adaptive Intelligent RTOS

AIROS is a simulated Real-Time Operating System (RTOS) kernel enhanced with a Q-Learning reinforcement learning scheduler. It dynamically adapts task priorities at runtime based on deadline history, urgency, and CPU utilisation.

GitHub Repository View Code ↗

Written in C11. Explores embedded systems & AI scheduling research on simulated ARM Cortex-M architecture.

Project Architecture

Built entirely from scratch, the AIROS kernel manages up to 16 concurrent tasks using a custom Task Control Block (TCB). The core features include:

  • Simulated context switching using ARM Cortex-M stack frames.
  • Inter-Process Communication (IPC) primitives: Mutexes (with priority inheritance), Semaphores, and Message Queues.
  • Deadline tracking and periodic task releases.
  • Three distinct scheduling policies: Fixed Priority, Round Robin, and the experimental Q-Learning RL agent.

Reinforcement Learning Scheduler

The real innovation lies in the RL scheduler. Instead of relying on static priorities set at compile time, the Q-Learning agent observes system state (deadline urgency, miss rate, burst ratio) and assigns priority dynamically using the Bellman equation.

The agent receives a +1 reward for meeting a deadline and a -1 penalty for missing it, alongside efficiency bonuses.

Benchmark Results

The system was simulated over 5,000 ticks with 8 heterogeneous tasks. The RL scheduler successfully converges to a hard real-time standard (<5% miss rate) after its initial exploration phase.

Scheduler Deadline Misses Deadline Hits Miss Rate
Fixed Priority 214 4,898 4.19%
Round Robin 123 4,939 2.43%
AIROS RL-Adaptive 242 4,919 4.69%

Note: The RL scheduler converges at tick 501. The higher initial miss count is due to early-epoch exploration (epsilon-greedy policy) as the agent discovers the optimal scheduling policy.
