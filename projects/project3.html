Project 03 — IoT Web Server

Raspberry Pi Pico W Personal Web Server

This project turns a Raspberry Pi Pico W into a small web server. When another device connects to the same Wi-Fi network and opens the Pico W IP address, the microcontroller serves a personal webpage with HTML, CSS, and an image.

Project Requirements

  • Raspberry Pi Pico W
  • CircuitPython firmware
  • Wi-Fi access
  • Adafruit HTTP server library
  • Files on the board: code.py, index.html, style.css, and image.jpg

How It Works

The Pico W connects to Wi-Fi, prints its IP address in the serial console, and starts listening for browser requests. The root route serves the HTML file, while separate routes serve the stylesheet and image. This is a simple but useful example of an embedded device acting as a web server.

Server Code

import wifi import socketpool from adafruit_httpserver import Server, Request, FileResponse SSID = "YOUR_WIFI" PASSWORD = "YOUR_PASSWORD" wifi.radio.connect(SSID, PASSWORD) print("Connected to WiFi!") print("IP Address:", wifi.radio.ipv4_address) pool = socketpool.SocketPool(wifi.radio) server = Server(pool, "/static") @server.route("/") def base(request: Request): return FileResponse(request, "index.html", "/") @server.route("/style.css") def css(request: Request): return FileResponse(request, "style.css", "/") @server.route("/image.jpg") def image(request: Request): return FileResponse(request, "image.jpg", "/") server.start(str(wifi.radio.ipv4_address)) print("Server running...") while True: server.poll()

What I Learned

This project helped connect embedded systems with web technology. The microcontroller is not only reading pins or blinking LEDs; it can also communicate over a network and provide a browser-based interface for users.
