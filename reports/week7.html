Week 07 — IoT Web Server

Personal Web Server on Raspberry Pi Pico W

This assignment introduced the Raspberry Pi Pico W as a networked embedded device. The goal was to serve a small personal website directly from the microcontroller using CircuitPython, Wi-Fi, sockets, and the Adafruit HTTP server library.

Learning Objectives

  • Connect a Pico W to a Wi-Fi network and read its IP address from the serial console.
  • Use a microcontroller as a small HTTP server.
  • Serve separate HTML, CSS, and image files from the board.
  • Understand request/response communication in a practical IoT example.
  • Reflect on the limits of microcontroller-hosted websites.

Project Structure

The required files are stored on the CircuitPython drive. The browser requests a route, and the Pico W responds with the correct file.

CIRCUITPY/ |-- code.py |-- index.html |-- style.css |-- image.jpg `-- lib/ |-- wifi |-- socketpool `-- adafruit_httpserver

Server Code

The server connects to Wi-Fi, creates a socket pool, then maps routes to files. Opening the board IP address in a browser loads the page from the microcontroller.

import wifi import socketpool from adafruit_httpserver import Server, Request, FileResponse SSID = "YOUR_WIFI" PASSWORD = "YOUR_PASSWORD" wifi.radio.connect(SSID, PASSWORD) print("Connected to WiFi!") print("IP Address:", wifi.radio.ipv4_address) pool = socketpool.SocketPool(wifi.radio) server = Server(pool, "/static") @server.route("/") def base(request: Request): return FileResponse(request, "index.html", "/") @server.route("/style.css") def css(request: Request): return FileResponse(request, "style.css", "/") @server.route("/image.jpg") def image(request: Request): return FileResponse(request, "image.jpg", "/") server.start(str(wifi.radio.ipv4_address)) print("Server running...") while True: server.poll()

IoT Protocol Notes

HTTP is simple and works well when a browser needs to request a page or API response from a device. MQTT is better for frequent low-power IoT messages because it uses a publish/subscribe model through a broker. CoAP and WebSockets are also useful in specific IoT cases, depending on power, bandwidth, and real-time requirements.

Reflection

In this setup, the microcontroller acts as a server and the browser acts as the client. The main limitation is that a small board has limited memory and processing power, so it is best for lightweight dashboards, status pages, and local control interfaces rather than heavy websites. The same idea can be used in real IoT systems to display sensor readings, control outputs, or configure a device from a phone or laptop.
