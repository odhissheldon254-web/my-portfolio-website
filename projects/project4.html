Project 04 — Sensors & Output

Sensor and Display System

This project documents a sensor-based embedded system that reads real-world input and presents the result through an output display. It connects the Week 4 and Week 5 work on Raspberry Pi Pico programming, sensor integration, analog interfacing, and LCD-style feedback.

Project Goal

The goal is to build a system where a microcontroller reads a sensor value, processes it in code, and displays or outputs the result in a way a user can understand. The same structure can be used for temperature, humidity, distance, light, or battery monitoring systems.

Possible Components

  • Raspberry Pi Pico or Pico W
  • DHT22 temperature and humidity sensor, ultrasonic sensor, potentiometer, or LDR
  • LCD or OLED display
  • Jumper wires and breadboard
  • CircuitPython or C firmware

Embedded Logic

A sensor produces a signal that represents a physical quantity. For analog sensors, the ADC converts voltage into a digital value. For digital sensors, communication may happen through a protocol such as I2C, SPI, UART, or a single-wire timing protocol. The code then formats the result and sends it to a display or another output.

# Pseudocode for a sensor display system initialize_sensor() initialize_display() while True: reading = read_sensor() value = convert_reading(reading) display_text("Value:", value) wait_before_next_sample()

Analog Interfacing Notes

ADC is used when a microcontroller needs to read a continuous voltage from the real world. PWM can create analog-like output for LED brightness or motor speed control by changing duty cycle. DAC, where available, produces a real analog voltage for signal generation or more precise control.

What I Learned

This project brings together the full embedded pattern of input, processing, and output. A reliable system needs correct wiring, suitable signal conversion, stable code, and output that makes the measurement easy to interpret.
